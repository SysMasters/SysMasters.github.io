AndroidStudio自定义模板

发表于 2019-11-02

1、模板文件结构

AndroidStudio中模板全部放在/plugins/android/lib/templates/中

  • template.xml(创建面板获取输入参数)
  • globals.xml.ftl(全局变量参数)
  • recipe.xml.ftl(主要用于生成我们实际需要的代码，资源文件等)
  • root(存放对应源码的ftl文件和资源文件)
  • template_blank_activity.png(模板效果预览图)

1.1、template.xml

使用到的函数${layoutToActivity(layoutName)}

作用是将Activity的名字转为规范的layout名，比如参数MainActivity就会返回activity_main。除此之外，还有许多常用的函数：

  • camelCaseToUnderscore(string)：驼峰命名风格转下划线
  • escapeXmlAttribute(string)：此函数用来转义字符串，例如 Android’s，它可以用作 XML 属性值：Android's。特别是，它将转义’，”，< 和 ＆
  • escapeXmlText(string)：此函数用来转义字符串，例如 A & B’s 可以将其用作 XML 文本。这意味着它将转义 < 和 >，但不像 escapeXmlAttribute 它将不会转义’ 和 “。在前面的示例中，它将转义字符串为 A & B\s。请注意，如果您想要使用 XML 文本作为 资源的值，您应该考虑使用 escapeXmlString，因为它执行额外的所需的字符串资源转义
  • escapeXmlString(string)：此函数用来转义字符串，例如 A & B’s ，它适合作为 XML 文本插入字符串资源文件中，例如 A & B\s 。除了转义 < 和 ＆ 之类的 XML 字符外，它还执行其他Android 特定的转义，例如使用反斜杠转义撇号
  • extractLetters(string)：此函数从字符串中提取所有字母，有效删除任何标点符号和空白字符
  • classToResource(string)：把Activity转换为资源标识符。比如MainActivity会转换为main
  • layoutToActivity(string)：将资源标识符转换为Java字符。比如activity_main会转换为“MainActivity”
  • slashedPackageName(string)：包名转路径。比如：com.example.foo返回com/example/foo
  • underscoreToCamelCase(string)：下划线转驼峰

1.2、globals.xml.ftl

全局变量文件，存放的是一些全局变量。可通过id获取到该值。如：${hasNoActionBar}

1.3、recipe.xml.ftl

标签说明：
  • ：从root中copy文件到我们的目录,比如我们的模板Activity需要使用一些图标，那么可能就需要使用copy标签将这些图标拷贝到我们的项目对应文件夹。
  • ：合并文件，比如将string.xml和并到我们的string.xml
  • ：和copy类似，但是可以看到上例试将ftl->java文件的，也就是说中间会通过一个步骤，将ftl中的变量都换成对应的值，那么完整的流程是ftl->freemarker process -> java。
  • ：生成后打开指定的文件，比如我们新建一个Activity，就会把这个Activity打开
  • ：依赖库，会自动下载指定的library

<#if >为FreeMarker条件表达式，可参考FreeMarker在线手册
